# martin_stock — 리바이프로덕트(REBYPRODUCT) 재고관리

제빵공장 재고·생산 관리 웹앱. **자체 SQLite DB가 데이터 원본**.
※ 2026-07-07 사용자 결정으로 엑셀 임포트 이력을 전부 비우고 **일일 입력으로 새로 작성 시작**.

## 실행

1. **`재고관리.exe` 더블클릭 (권장)** — Python 불필요. 서버 콘솔창이 뜨고 브라우저 자동 오픈.
   콘솔창을 닫으면 종료. DB는 exe와 같은 폴더의 `martin_stock.db`.
2. `실행.bat` 또는 `python app/main.py` — Python 환경에서 실행.

## 사내 서버로 쓰기 (다른 PC에서 접속)

1. 최초 1회: `서버_방화벽허용.bat`을 **관리자 권한으로 실행** (8600 포트 허용)
2. `재고관리.exe` 실행 → 콘솔창에 표시되는 `다른 PC에서 접속: http://IP:8600` 주소를 공유
3. 다른 PC/휴대폰은 브라우저로 접속 (설치 불필요)
- 서버 PC는 공유기에서 IP 고정(DHCP 예약) 권장. 콘솔창을 닫으면 전체 접속 종료.

## 로그인 / 권한

- 최초 계정: **admin / 1** — 반드시 비밀번호 변경 (사이드바 [비밀번호])
- **admin**(전체) / **op**(시급·단가·노무비 제외 전체) / **guest**(보기 전용)
- 사용자 추가·삭제·권한 변경: admin → 기준정보 관리 > 사용자 탭 (권한 변경은 접속 중인 화면에도 즉시 적용)
- exe 재시작 시 재로그인 필요 (세션은 서버 메모리)

## 구성

```
martin_stock/
├── 재고관리.exe             # 실행 파일 (배포본)

├── 서버_방화벽허용.bat       # LAN 접속 허용 (최초 1회, 관리자 실행)
├── 실행.bat                 # 개발용 실행
├── MartinStock.spec         # exe 빌드 설정
├── 백업/                    # DB 백업 (일일기록전체삭제전 = 과거 엑셀 이력 복구용)
├── doc/                     # 원본 엑셀 (참고용) + 배합비/ + 배합비_통합정리.xlsx
└── app/
    ├── main.py              # FastAPI 서버 + API
    ├── db.py                # 스키마 (+마이그레이션)
    ├── static/              # 프론트엔드 (index.html + app.js + vendor)
    ├── import_bom.py        # doc/배합비 엑셀 → bom 테이블 (재실행 = 배합비 전체 교체)
    ├── export_bom.py        # bom → doc/배합비/배합비_통합정리.xlsx
    ├── import_excel.py      # (구) 엑셀 전체 임포트 — ⚠️ 실행 금지: DB 초기화됨
    ├── update_from_excel.py # (구) 엑셀 증분 추가 — ⚠️ 실행 시 과거 이력 재유입
    └── import_stock_0707.py # (1회성) 7/7 실사 재고 재설정 — ⚠️ 재실행 시 7/7 기록 다시 교체됨
```

## 화면

- **대시보드**: 부족 자재 KPI/표, 생산·출고 추이, 소비기한 임박 LOT
- **생산 현황**: 일/주/월/년 집계 + KPI 요약 칩(계획·생산·달성률·불량률·금액) + 차트
  + 5섹션 보고서(자재 소모·인원가동·재고·특이사항) + 불량 사유 표시 + 표 검색 + 엑셀(CSV)
- **출고 현황**: 일/주/월/년 · 일별 출고 추이 차트 + 거래처 비중 도넛 · 제품별·거래처별 출고 집계
  + 개별 내역(생산 LOT·소비기한 D-day·금액) + 검색 + 엑셀(CSV)
- **일일 입력**: 달력 → 하루치 폼. 생산실적에 배합 입력 = 계획·자재 사용 자동 채움,
  출고는 생산일자(LOT) 선택, 원부자재 입고(유통기한·발주 자재 자동 제안), 인원별 투입 시간,
  각 카드에 🔍 검색해서 행 추가.
  · **📋 어제처럼**: 직전 생산일의 제품·라인·인원 구성 불러오기(수량은 비움)
  · **📋 예상 자재 소요·부족**: 생산 계획 × 배합비로 필요 자재량 집계 → 현재고 대비 부족 경고
  · **불량 사유**: 불량 수량 입력 시 사유(태움·이물 등) 기록 · **📷 생산 현장 사진** 첨부
  · 저장 시 **요약 확인 팝업**(생산·출고·자재·인원·특이사항 건수)
- **LOT 관리**: 생산일자별 완제품 재고 LOT — 소비기한(생산일+소비일)·D-day·임박/만료 경고,
  급한 순(만료→임박) 정렬 · 제품 썸네일 · 재고금액 · 만료 LOT 일괄 폐기,
  LOT 폐기 처리(재고 차감, 이력·취소). 재고 = 기초 + 생산 − 출고 − 폐기
- **제품 이미지**: 제품명 클릭 팝업에서 이미지 추가/변경/삭제 → exe 옆 `Image/` 폴더에
  제품명 파일로 저장(예: `Image\달광도넛 딸기 (45 g).png`). 목록·팝업에 썸네일 표시
- **기준정보 관리**: 완제품/원재료/부재료/배합비(반죽·토핑, 분할 무게→수율 계산)/거래처/인원/생산라인/사용자
  — 이름 검색, 제품명 클릭 = LOT·생산이력 팝업, 자재명 클릭 = 입출고 이력
  · **📋 미등록 점검 배너**(단가·소비일·안전재고 등 미등록 건수, 칩 클릭 = 미등록만 보기)
  · **⚡ 빠른 편집**(단가·소비일·안전재고·시급을 표에서 바로 입력, 즉시 저장) · 목록 엑셀(CSV)
- **분석**: 기간·집계 선택, 추이 차트, 제품 TOP(생산/출고), 불량 분석(불량률·사유별 집계),
  **원가·수익성**(배합비×자재단가+개당 노무비 → 마진율, 제품 클릭=자재별 구성, admin),
  재고 회전·장기재고(검색, CSV)
- **기록 조회**: 달력 + 날짜 상세(생산·불량사유/출고/입고/자재사용/인원가동/LOT/사진/특이사항)
  + 품목 검색(기간 지정 가능, 합계·CSV) · [전체] = 제품명 없이 기간 내 모든 제품 생산·출고 기록
- **접속 인원 / 채팅**: 우측 하단에 현재 접속 인원(🟢 N명) 표시, 클릭 시 접속자 목록 + 사내 채팅(폴링 기반, LAN 내부)

## exe 재빌드 (개발)

```
python -m PyInstaller --noconfirm MartinStock.spec
```
→ `dist\재고관리.exe`를 루트로 복사 후 **dist/build 폴더 삭제**.
⚠️ exe는 자기 옆의 martin_stock.db를 사용 — dist 안의 exe를 실행하면 빈 DB가 생깁니다.
