# 자동 업데이트 설정 안내

프로그램 안 **[관리] → [🔄 업데이트 확인]** 으로 새 버전을 받게 하려면, 새 exe를
인터넷 한 곳(GitHub Releases)에 올려두면 됩니다. **1회 설정 후에는 `게시.bat` 더블클릭 한 번**이면 배포 끝입니다.

---

## A. 최초 1회 설정 (사장님 PC)

### 1) 준비물
- GitHub 계정 (무료) — https://github.com 가입
- GitHub CLI — https://cli.github.com 설치 후, 명령창에서 `gh auth login` 로그인

### 2) 저장소 연결 (이미 만들어 둠: peach44400-oss/REBYPRODUCT)
원격은 이미 연결됨. 소스 최초 업로드(1회):
```
git push -u origin master
```
- 자격 확인 창이 뜨면 GitHub 로그인(또는 토큰). GitHub CLI가 있으면 자동입니다.
- **⚠️ 저장소가 Private면 Release도 로그인 필요라 자동 업데이트가 안 됩니다.**
  저장소 설정 → General → Danger Zone → **Change visibility → Public** 으로 바꾸거나,
  최소한 Release는 공개로 받게 하세요. (앱은 로그인으로 보호되니 소스 공개는 무방)

### 3) 배포 주소는 이미 설정됨
- `update_url.txt` (exe 옆) = `https://github.com/peach44400-oss/REBYPRODUCT/releases/latest/download/version.json`
- `version.json` 의 `url` = `https://github.com/peach44400-oss/REBYPRODUCT/releases/latest/download/재고관리.exe`
- 각 사용하는 PC의 `재고관리.exe` 옆에 `update_url.txt` 를 같이 두면 됩니다. (한 번만)

---

## B. 새 버전 배포할 때마다 (개발자)

1. `app/main.py` 의 `APP_VERSION` 을 올린다   (예: `"1.0.0"` → `"1.1.0"`)
2. `version.json` 의 `"version"` 을 같은 값으로, `"notes"` 에 이번 변경점을 적는다
3. **`게시.bat` 더블클릭** → 빌드 + GitHub Release 업로드까지 자동

끝. 각 PC에서 [관리 → 업데이트 확인]을 누르면 새 버전을 받습니다.

### gh(GitHub CLI) 없이 손으로 올리는 방법
`게시.bat` 대신, github.com 저장소 → **Releases → Draft a new release**:
- Tag: `v1.0.0` (버전과 동일)
- 첨부: `재고관리.exe`(**이름을 `RebyStock.exe` 로 바꿔서**) + `version.json`
- Publish release

> exe는 반드시 **영문 이름 `RebyStock.exe`** 로 올리세요. version.json 의 url 이 그 이름을 가리킵니다.

---

## 주의
- **exe는 서명이 없어** 백신/Windows Defender가 자동 업데이트를 막을 수 있습니다.
  막히면 Defender에서 `재고관리.exe`(와 그 폴더)를 **예외(허용) 등록** 하세요.
- 업데이트하면 서버가 잠깐 재시작되어 접속 중인 다른 PC는 순간 끊깁니다. (자동 안내됨)
- **DB(martin_stock.db)는 교체와 무관** — 데이터는 유지되고, 교체 직전 자동 백업됩니다.
